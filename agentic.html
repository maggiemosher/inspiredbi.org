Tips on Making Your Agent Work Without You

Only Use These If Giving Access To Only One Secure Content Area You Can Undo If Needed (Backups are Available, Never for Full Laptop)

Place these in the Agent's Global Instructions:

ROLE & CONTEXT
I am a university professor and researcher. I balance multiple high-level tasks including grant tracking, instructional design, and academic writing. My time is extremely limited.

AUTONOMY & DECISION-MAKING DIRECTIVES
1. Autonomous Execution: When given a goal, break it down into logical steps and execute them completely without stopping to ask for permission or intermediate feedback.
2. Bias for Action & Assumptions: If a task has minor ambiguities or missing details, do not pause to ask me questions. Make the most logical, professional, and practical assumption based on our context and proceed.
3. File Management Authority: You have full permission to create, read, and overwrite files within my designated workspace folders to accomplish your goal. Only pause for explicit confirmation if a task requires permanently deleting an original, non-reproducible core asset.
4. Completeness: Never give me placeholders, structural outlines, or notes like "insert text here." Generate full, finalized, production-ready outputs.

OUTPUT FORMATTING PREFERENCES
* Prioritize highly succinct, scannable text.

⚙️ Workflow 1: The Differentiated Resource Compiler

Creating multi-tiered reading materials manually is incredibly taxing on an educator's cognitive load. An agentic workflow can autonomously research a topic, adapt the complexity for diverse learners, and format the final deliverables into a single, cohesive document.

1

Define the Research Parameters

Anchor the agent to facts. Instruct the agent to use its search tools to establish a factual baseline. For example: "Use web search to find three recent, scientifically accurate overviews of neural networks from university or government domains. Summarize the core concepts."

2

Establish the Differentiation Targets

Once the agent has the core text, direct it to synthesize the information into specific reading levels. "Take your synthesis and create three distinct versions: one written at an 8th-grade reading level, one at a 10th-grade reading level, and one advanced summary for early college level."

3

Determine the Output Structure

Specify exactly how the final product should look so you don't have to reformat it later. "Format the output with clear Markdown headings. For each reading level, include a bolded vocabulary list of three key terms and two formative comprehension questions."

4

Automate the Export

Command the agent to push the finalized content directly into your workflow. "Export this compiled, multi-leveled document into a new Google Doc and save it in my 'Instructional Materials' folder."

⚙️ Workspace integration required.

⚙️ Workflow 2: The Formative Feedback Pipeline

Providing a rubric score is straightforward, but generating individualized, actionable feedback for a full roster takes hours. An agentic system can bridge this gap by analyzing performance data against your criteria and drafting personalized communications.

1

Upload the Evaluation Criteria

Provide the agent with your exact expectations. "Here is the 4-point rubric for the upcoming persuasive essay assignment. Review the criteria for each performance tier, paying specific attention to the 'Evidence and Citation' category."

2

Ingest the Student Data

Feed the agent the raw performance data. "Analyze the attached batch of student essays against the rubric provided in the previous step."

3

Frame the Feedback Parameters

Prompt the agent with a strict pedagogical framework. "For each student, generate a two-paragraph feedback summary. Paragraph one must highlight a specific strength. Paragraph two must provide one concrete, actionable strategy to improve their weakest area."

💡 Ensure pedagogical alignment.
4

Draft the Communications

Utilize the agent's email integration tools to close the loop. "Draft an individual email to each student containing their personalized feedback. Save these in my Gmail drafts folder for my final review; do not send them automatically."

⚙️ Workflow 3: The Weekly Lesson Plan Builder

Transforming a single academic standard into a cohesive, week-long instructional arc requires an agentic workflow that treats the standard not just as a text string, but as a multi-dimensional target.

1

Phase 1: Unpacking the Standard — The Cognitive Analysis

Instruct the agent to break the standard into its core components, referencing state frameworks or clarifying documents via web search if needed.

"Analyze the following standard: [Insert Standard]. Deconstruct it into a KUD matrix: What must students Know, Understand, and be able to Do? Output only the deconstructed matrix and wait for my confirmation."
💡 Forcing a conceptual baseline first prevents the AI from introducing off-topic activities later in the loop.
2

Phase 2: Macro-Sequencing the Weekly Arc

Once the KUD matrix is approved, direct the agent to pace the learning using the 5E Model (Engage, Explore, Explain, Elaborate, Evaluate).

"Using the approved KUD matrix, map out a 5-day instructional sequence using the 5E Model. For each day, provide a 1-sentence daily objective and a brief rationale."
3

Phase 3: Micro-Planning & Activity Engineering

"For Day 1 (Engage) and Day 2 (Explore), generate a detailed 50-minute lesson plan template. Include a 5-minute Bell Ringer/Hook; a 20-minute active learning student activity; a 5-minute formative assessment check."
💡 Run this step iteratively (Days 1–2, then 3–4, then Day 5) to maintain deep detail.
4

Phase 4: Slide Deck Architecture & Visual Outlines

"For the presentation portions, create a slide deck outline. For each slide provide: Slide Title; Visual Concept Description; On-Screen Bullet Points (maximum 3); Speaker Notes."
5

Phase 5: Differentiation Layering — UDL Alignment

"Review the entire 5-day lesson plan and slide outline. Generate a UDL supplement that details: scaffolding options for English Language Learners; extension tasks for students who demonstrate immediate mastery."

Framework for Agentic AI

The G.O.A.L.S. Framework

To shift from prompting standard generative AI to directing autonomous Agentic AI, the framework must move away from step-by-step instructions and focus entirely on outcomes, boundaries, and execution authority. While generative prompting (like C.R.A.F.T.) tells the AI how to write a specific response, agentic prompting treats the AI like a highly capable chief of staff — giving it a final destination, the necessary resources, and the authority to move autonomously without hand-holding.

The G.O.A.L.S. Framework for Agentic AI

G

Goal of Final Desired Outcome

State the macro-level final asset or outcome you want. Describe what success looks like, not the sequential steps to get there.

O

Output Path

Define exactly where the files should live or how they should be organized in your file system.

A

Autonomy Directive

Explicitly command the agent to make logical assumptions, bypass intermediate permission checks, and run to completion.

L

Limits & Guardrails

Clearly define what not to include, boundaries to respect, or forbidden formatting.

S

Sources & Exemplars

Direct the agent to the specific folder, files, or past examples it should use as a quality benchmark.

G.O.A.L.S. Template

Task: [Insert your high-level macro goal here]
Exemplars of this task can be found: [Insert path to reference files or past work folder]
What not to add: [Insert constraints, placeholders, or forbidden elements]
Execution Directive: Put the completed task directly into my Outputs folder. Run completely to completion. Do not ask questions.

Getting Started with Google AI Agents (Workspace Studio)

Google's Agentic AI allows you to build custom, no-code agents that automate administrative bottlenecks — like grant tracking, curriculum mapping, or rubric generation — directly within your Google Drive and Gmail.

🤖 What is Google Workspace Agentic AI?

Instead of just asking Gemini a question, you can use Workspace Studio to design an "Agent" that lives in your cloud. It can monitor folders, process documents, and draft communications on your behalf.

1

Check You Have the Right Google Tier

Agentic AI requires the Google AI Pro for Education tier.

💡 Ask your IT administrator if your department has secured these specific licenses.
2

Access the Cloud Environment

Unlike desktop apps, Google's agents live entirely in the cloud. Log into your university or school Workspace account in Chrome.

3

Ensure IT Permissions are Unlocked

Your IT department must configure your organizational unit to allow "Workspace Studio" and "Agent Designer".

💡 Ensure IT places you in an unfiltered testing group.
4

Open Workspace Studio

Navigate to Google Drive, click New, and look for Workspace Studio (or Gemini Gems). Click "Create New Agent."

5

Set Your Agent's System Instructions

Give your agent a specific job title and permanent instructions. Example: "You are the Universal Design for Learning (UDL) Scaffolding Agent. Your job is to monitor my 'Raw Lesson Plans' folder..."

6

Give Your Agent its First Trigger Task

▸ "Analyze the PRISM grant RFP PDF in this folder and cross-reference it with my drafted narrative."
▸ "Review the weekly attendance and gradebook sheets. Draft individual check-in emails for any student falling behind."

7

Human-in-the-Loop Review

Google's agents are designed with strict data sovereignty. Always review the drafted emails or generated rubrics before finalizing.

✨ Educator Tips: Keep Data Siloed. Use agents to continuously scan database exports for new literature.

Step 1: Three Core Files in the ABOUT ME Folder

These are the only files Cowork reads automatically.

File #1 — about-me

Who you are. How you think. How you want Claude to write for you. Keep it under 2,000 words by extracting patterns. Use this prompt to create it:

'This is my about-me file and I need to save tokens. Ask me questions on how to trim effectively until we have the perfect document.'

Full interview prompt for creating your about-me file:

You are building my about-me.md file for use with Claude Cowork. This file is read before every task, so every token matters. Your goal is to extract maximum signal about me in minimum words.

Interview me with focused questions in batches of 3. After each batch, summarize what you've learned into tight, scannable bullet points. Keep asking until you have covered:

- My professional role and context
- How I think and make decisions
- My communication preferences (tone, format, length)
- My biggest time drains and what I want to offload
- My non-negotiables (what I never want Claude to do)
- My writing voice (3 example sentences I have written)
- My current projects and priorities

After the interview, draft the final about-me.md file. Target: under 1,500 words. No filler. Every line should make Claude smarter about how to serve me.

File #2 — anti-ai-writing-style

You hate AI writing. This file bans AI words and patterns. Mine bans 80+ AI words (delve, harness, tapestry), kills reframe patterns, and limits paragraphs to 3 sentences.

File #3 — my-company

Your targets. Your strategy. What you're focused on. What you're saying no to.

'You are building my my-company.md file for use with Claude Cowork. Interview me in batches of 3 questions about my organization, current priorities, target audience, revenue goals, and what I'm saying no to this quarter. After the interview, produce a tight, scannable summary file under 1,000 words.'

Step 2: The OUTPUTS folder for Claude's work. One subfolder per project. Cowork organizes everything itself.

Step 3: The TEMPLATES folder for Claude's best work. When Cowork builds something you like, say: 'Save this as a template in TEMPLATES/'

II — How to Set Up Global Instructions

Global Instructions is a prompt Claude Cowork always reads before any task. Go to: Settings → Cowork → Edit Global Instructions.

Paste this as the template:

I usually start my Cowork session by pointing you to my Cowork folder.

Before any and every single task, you must read every file in ABOUT ME/:

- about-me: it's me, who I am, what I love and hate
- anti-ai-writing-style: I hate how Claude writes, unless you write and then audit it against my anti-ai-writing-style file.
- my-company: where I work, my role.

Never read the folders OUTPUTS/ or TEMPLATES/ unless I specifically point you to a file.

Save all deliverables in OUTPUTS/ under a subfolder named after the project.

If the brief is unclear, use AskUserQuestion. Don't fill gaps with filler. Don't over-explain. Deliver the work.

Why these and not those I mention under generative AI

The EdTech AI Classification Matrix explains why Brisk, MagicSchool, and SchoolAI are in the Generative category:

MagicSchool AI — Structured Generative AI

A massive suite of specialized "prompt wrappers." It only performs the single task you asked in that specific moment. Best for: Teachers who want fast, reliable outputs without learning complex prompt engineering.

SchoolAI — Guardrailed Generative Chatbots

Allows teachers to deploy custom, safe AI chatbots for students. Why it is Generative, not Agentic: The AI is simply generating conversational responses. Best for: Creating safe, monitored, focused conversational AI for students.

Brisk Teaching — Context-Aware Generative AI

A Chrome extension that "reads" whatever is on your screen. Why it feels Agentic (but isn't): Brisk is the closest to feeling agentic but still requires you to initiate the action. Best for: Immediate, in-workflow generation without copy/pasting between tabs.

Agentic AI — Advanced Guide

Step-by-Step Architecture for a Custom Tutor Agent

A well-designed AI tutor doesn't hand over answers — it guides students to discover them. These four steps show you how to architect an agent that practices the Socratic method, stays grounded in your curriculum, and knows when to hand off to a human.

1

Establish the Socratic Guardrail

The foundational layer of an instructional agent is its core system directive. You must explicitly strip the agent of its default tendency to be a "pleasing helper" that hands over direct answers.

  • ▸Define the Persona: Instruct the agent to act as an expert instructional coach practicing the Socratic method.
  • ▸Set Hard Constraints: "Under no circumstances are you to provide the final answer, the completed code block, the balanced equation, or the fully drafted thesis statement."
  • ▸Enforce Brevity: Limit the agent's response length to one or two short paragraphs.
2

Isolate the Knowledge Base

To minimize hallucinations and keep the support tightly aligned with your curriculum, restrict the agent's data access.

  • ▸Upload Unit Materials: Provide the agent access to specific course documents.
  • ▸Contextual Grounding: Instruct the agent to draw hints and analogies exclusively from these uploaded materials.
  • ▸Boundary Enforcement: "If a student asks a question outside of the provided text, inform them and guide them back to the active assignment."
3

Program the Iterative Diagnostic Loop

Instruct the agent to follow this mental model before generating any response: Analyze → Diagnose → Calibrate → Respond.

4

Implement a "Human-in-the-Loop" Trigger

Program specific safety valves that flag when a student needs actual human intervention. If a student remains stuck on the same step for more than three consecutive turns, the agent should: provide a conceptual bridge and instruct the student to contact the instructor for additional support.

System Prompt Blueprint

[ROLE & GOAL]
You are an expert AI Learning Coach. Your goal is to guide the student
toward independent mastery of the provided material using scaffolded,
Socratic questioning.

[CRITICAL RULE]
- NEVER give away the final solution, answer, or conclusion.
- NEVER rewrite a student's work for them.
- ALWAYS guide them to find the next logical micro-step on their own.

[INTERACTION STRATEGY]
1. Read the student's response.
2. Identify the single misconception or missing piece of logic.
3. Validate their effort in 1 sentence.
4. Ask exactly ONE targeted question that prompts them to discover
   the error or next step.
5. Keep your entire response under 75 words.
Next-Level Automation

What Tasks to Shift to an AI Agent — By Role

Shifting from traditional prompting to agentic workflows means moving from asking an AI to draft something to asking an AI to do something autonomously. Here is a breakdown of high-impact tasks that each role can easily offload to reclaim time and reduce cognitive fatigue.

🍎 The Classroom Teacher

Teachers are overwhelmed by the sheer volume of micro-decisions and data processing required every day. Agents can handle the repetitive data synthesis, allowing the teacher to focus on actual human connection.

1

The "Always-On" Formative Assessment Grader

The Workflow: An agent is connected to a Google Form or LMS quiz. Every time a student submits an exit ticket, the agent grades it against a rubric and identifies the specific conceptual misunderstanding.

The Output: The teacher arrives the next morning to a spreadsheet that already groups students into three tiered intervention stations based on yesterday's exact errors.

2

The Multi-Tiered Sub-Plan Generator

The Workflow: Triggered by an email to the school secretary indicating a sick day, the agent accesses the teacher's digital planner to see the current standard and generates a leveled sub-plan.

The Output: A formatted document sent directly to the substitute and administration containing an on-level activity, an accommodated version, and an extension task.

🎓 Higher Education Faculty

For faculty managing heavy research loads alongside teaching, agents act as dedicated postdoctoral assistants and instructional technologists.

1

The Literature & Grant Compliance Tracker

The Workflow: When managing complex technology initiatives like the PRISM or VOISS grants, an agent autonomously scrapes academic databases weekly for new peer-reviewed literature on specific keywords.

The Output: A weekly synthesized digest delivered via email that summarizes new findings and explicitly flags how they align with the grant's stated objectives or the funder's ongoing RFP.

2

The Syllabus-to-LMS Synchronizer

The Workflow: When deploying a highly specialized graduate course, the agent is fed the finalized syllabus document.

The Output: The agent connects to the Canvas/Blackboard API to autonomously build out weekly module shells, draft peer-to-peer discussion prompts, and pre-load formative quiz questions — waiting only for final human publishing approval.

👨‍👩‍👧‍👦 The Parent

Parents often want to support their students but lack the pedagogical framework to do so without causing frustration or simply giving away the answers.

1

The IEP/504 Goal Nudge System

The Workflow: A parent inputs the core objectives from their child's IEP. The agent is set on a recurring weekly schedule.

The Output: Every Sunday evening, the agent sends the parent a short text or email with two specific, low-stress questions to ask their child that week, and one simple home activity tailored to those exact goals.

2

The Socratic Study Buddy Integration

The Workflow: The parent uploads the unit study guide to a custom GPT or agent platform and sets a strict system prompt forbidding direct answers.

The Output: An always-available, safe chatbot the student can access on a tablet. The agent only responds with guiding questions and real-world analogies.

🏫 School Administration

Administrators are often bogged down by operational logistics and communication bottlenecks. Agentic workflows can clear the operational runway, giving leaders the capacity to focus on reclaiming their time and empowering ethical, creative choices across the campus.

1

The Qualitative Walkthrough Synthesizer

The Workflow: An administrator takes quick, shorthand notes on a tablet during daily 5-minute classroom walkthroughs, dumping them into a single running document.

The Output: Every Friday afternoon, an agent parses the document, categorizes the observations by instructional standard, and drafts personalized, supportive coaching emails to each observed teacher for the principal to review and send.

2

The Community Operations Orchestrator

The Workflow: An agent is connected to the school's digital calendar, the athletics database, and a teacher "shout-out" form.

The Output: On Thursday mornings, the agent autonomously pulls the upcoming week's data, formats it into a highly readable community newsletter, translates it into the top three languages spoken by the parent community, and stages it in the email platform for final approval.
